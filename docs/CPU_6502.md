# 2A03 CPU Milestone

The `cpu-6502` crate implements the documented NMOS 6502 instruction set as
used by the NTSC Ricoh RP2A03, with decimal arithmetic disabled while the D flag
remains readable and writable.

## Implemented with generated and independent single-step tests

- The canonical set of 151 documented opcode encodings returns explicit
  mnemonic, addressing mode, base cycles, instruction length, and page-cross
  policy.
- A committed 190-vector architectural sample curated from the MIT-licensed
  `SingleStepTests/65x02` RP2A03 suite checks all 151 documented encodings. It
  compares final PC, SP, A, X, Y, status, declared RAM, and instruction cycles;
  it also includes both cycle profiles for all 23 page-penalty encodings and
  2/3/4-cycle profiles for all eight branches.
- The identity-checked Kevin Horton `nestest` V1.00 trace passes all 8,991 rows
  and 8,990 transitions through mapper 0. The supported set includes the exact
  76 stable undocumented encodings exercised by that fixture: NOP aliases,
  LAX, SAX, DCP, ISC, SLO, RLA, SRE, RRA, and the `$EB` SBC alias.
- Three project-owned NROM diagnostics match 47-row architectural traces
  generated by pinned BSD-3-Clause py65 through the real parser, mapper-0 bus,
  runner, and CLI. They cover CPU RAM mirrors, PRG mapping/RAM, ignored ROM
  writes, trainer endpoint preload, stack and control flow, branches, and
  page-cross cycles; reset and bus order are not checked.
- Instruction-level architectural state and cumulative cycle traces.
- Immediate, zero-page, indexed, absolute, indirect, indexed-indirect,
  indirect-indexed, accumulator, implied, and relative addressing.
- Wrapping zero-page pointers and the NMOS `JMP ($xxFF)` high-byte wrap quirk.
- Binary ADC/SBC carry, borrow, overflow, zero, and negative behavior.
- Stack order and architectural effects for JSR/RTS, BRK/RTI, PHP/PLP, and
  PHA/PLA.
- Branch taken and page-cross cycle increments.
- Reset-vector power-on state with seven recorded cycles.
- Failure-atomic exhaustion at the final seven cycle-counter values. `step`
  reserves the supported eight-cycle maximum before touching the bus, so it
  may stop conservatively even when a shorter instruction would numerically fit.

## Explicitly not yet claimed

- Per-cycle bus access order, dummy reads/writes, or read-modify-write double
  writes.
- IRQ/NMI line sampling, one-instruction I-flag latency, interrupt hijacking,
  reset bus cycles, DMA stalls, RDY behavior, or scheduler integration.
- Undocumented opcodes outside the exact 76-encoding `nestest` set, including
  jam and hardware-sensitive unstable encodings.
- Exhaustive input-state coverage. The pinned single-step sample is independent
  evidence across every documented encoding, not all 10,000 upstream scenarios
  per opcode.

Those limitations prevent this crate from being called cycle-accurate or a
complete NES CPU. It is a trace-first architectural layer with independent
instruction-boundary and full mapper-trace evidence that the bus and interrupt
milestones will refine. After exact identity verification, the strict CLI
allows writes to `$4004`-`$4007` and `$4015`; the reviewed log makes exactly
five terminal writes there. This is not APU emulation, and the normal mapper bus
still faults unimplemented I/O.
